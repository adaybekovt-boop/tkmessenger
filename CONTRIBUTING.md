# 🤝 Инструкция для разработчиков

## Правила кодинга
- Используйте TypeScript для всех новых файлов.
- Компоненты фронтенда должны лежать в `client/src/components`.
- Бизнес-логика бэкенда должна быть вынесена в `server/src/services`.

## Процесс работы
1. Создайте ветку от `develop`: `git checkout -b feature/your-feature`.
2. Внесите изменения.
3. Запустите линтинг: `npm run lint`.
4. Сделайте Pull Request в ветку `develop`.

## Тестирование
- Перед пушем запустите тесты: `npm run test`.
- Все новые функции должны сопровождаться unit-тестами.
